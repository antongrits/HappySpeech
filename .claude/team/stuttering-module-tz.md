## Целевая аудитория

Дети 5–8 лет с **заиканием** (логоневрозом, тонико-клоническим) лёгкой и средней степени тяжести,
находящиеся на начальной стадии коррекции. Тяжёлые формы заикания (судороги, сопутствующие
движения, стойкий логофобический компонент) — индикатор для направления к очному специалисту.

Приложение **НЕ** заменяет логопеда. Роль модуля — дополнительная домашняя практика в промежутках
между очными занятиями, закрепление навыков плавной речи в безопасной игровой среде.

## Методологическая основа

### Fluency Shaping (FS) — техника плавной речи

Основной подход, адаптированный для детей дошкольного и младшего школьного возраста.
Оригинальная методика (Van Riper, Webster) разработана для взрослых — HappySpeech реализует
упрощённую, игрофицированную версию её ключевых компонентов:

- **Замедленный темп** — ребёнок говорит медленнее обычного (75–90 BPM vs нормативных 120–180 BPM).
  Замедление снижает вероятность спазма и даёт время на моторное планирование слога.
- **Мягкий голосовой старт (soft onset)** — голос включается плавно, без резкого напряжения гортани.
  Особенно важен для тонического компонента заикания, при котором именно начало слова провоцирует блок.
- **Плавные переходы между звуками** — коартикуляция без резких разрывов внутри слова.
  Обеспечивается замедленным темпом и удлинёнными гласными.
- **Диафрагмальное дыхание** — речь строится на плавном управляемом выдохе.
  Вдох — спокойный, носовой, 2–3 сек; выдох — медленный, ртом, 4–6 сек, в течение которого
  произносится речевой материал.

### Stuttering Modification (SM) — для данного возраста НЕ применяется

Техники отмены (cancellation), выхода (pull-out) и подготовительного набора (preparatory set)
требуют развитой метакогнитивной рефлексии — способности осознать момент заикания и
намеренно его изменить. У детей 5–8 лет эта способность недостаточно сформирована.
SM вводится логопедом не раньше 9–10 лет при наличии осознанного отношения ребёнка к проблеме.

### Метроном-метод

Внешняя ритмическая подсказка: ребёнок синхронизирует слоги с равномерным тактовым сигналом.
Метроном устраняет неопределённость момента начала и длительности слога — именно эта
неопределённость провоцирует тонический блок. Эффективен при клоническом компоненте,
менее эффективен при чисто тоническом, но безвреден как подготовительный инструмент для любой формы.

Рекомендуемый темп для начала работы: 60–80 BPM. Норма речи — 120–180 BPM. Разрыв намеренный:
медленный темп снижает нагрузку на моторное планирование.

### Визуальная биообратная связь

Ребёнок видит на экране реакцию объекта (дерево, облако, шарик) на собственный голос или выдох.
Это даёт внешний «якорь» для управления громкостью, длительностью и плавностью — без
метаязыка взрослого («говори тише», «не торопись»).

Технический механизм: AVAudioEngine RMS-измерение, тот же паттерн что в `BreathingInteractor`:
`handleAmplitude(_:)` → `adaptiveThreshold` → objectScale / progress.

## 4 упражнения модуля

### Упражнение 1. Метроном-говорилка («Шагай за словом»)

**Педагогическая цель:** сформировать навык послогового произнесения слов в замедленном темпе
под внешний ритмический ориентир. Снизить частоту тонических блоков за счёт предсказуемости
момента начала каждого слога.

**Целевой навык по FS:** замедленный темп + плавные переходы между слогами.

**Игровой сценарий:** Ляля идёт в поход. Каждый шаг — один слог слова. Ребёнок помогает Ляле
«прошагать» слово до конца дорожки.

**Механика:**

- Экран: анимированная дорожка с делениями (по числу слогов в слове).
- Метроном: тактовый сигнал (звук щелчка) каждые N мс; N зависит от уровня сложности.
  Easy: 800 мс (75 BPM), Medium: 667 мс (90 BPM), Hard: 571 мс (105 BPM).
- Ребёнок произносит **один слог** при каждом тике.
- Микрофон: AVAudioEngine RMS. Если амплитуда выше adaptiveThreshold в течение ≥100 мс
  после тика — слог засчитан, персонаж делает шаг, деление закрашивается.
- После прохождения всего слова — анимация награды (Ляля добежала до финиша), 3 звезды за плавность.
- Сессия: 8–12 слов за раунд.

**Контент:** существующие units из контент-паков (sound_sh_pack, sound_r_pack и др.),
фильтр по слоговой структуре. Easy: 1–2 слога (кот, шар, рыба). Medium: 3 слога (машина, собака).
Hard: 4+ слога (черепаха, карандаш, электричка).

**Адаптивная сложность:** `AdaptivePlannerService` выбирает уровень на основе fluency rate
за последние 3 сессии. Начальный уровень — Easy для всех новых пользователей модуля.

**Метрика успеха:** ≥75% слогов засчитаны в пределах окна тика (±200 мс). При ≥80% — переход
на следующий уровень сложности.

### Упражнение 2. Дыхательная регуляция («Длинный выдох»)

**Педагогическая цель:** развить диафрагмальное дыхание с управляемым длинным выдохом —
физиологическую базу плавной речи. Без правильного дыхания FS-техники не работают.

**Целевой навык по FS:** диафрагмальное дыхание, длинный плавный выдох 4–6 секунд.

**Игровой сценарий:** Ляля поливает волшебное дерево. Чем дольше ребёнок дует — тем больше
листьев распускается. Если остановится рано — дерево засыхает.

**Механика:**

- Reuse: полный паттерн `BreathingInteractor` (state machine idle → tutorial → warmUp → playing
  → success/failure → summary). Меняется только сцена (`.tree` вместо `.dandelion`) и целевые
  пороги.
- RMS-детекция: непрерывный выдох ≥3 сек выше adaptiveThreshold = baseline × 1.5.
- При прерывании выдоха (амплитуда ниже порога дольше 300 мс) — анимация листьев
  останавливается, начинаем заново.
- Goal: 5 успешных выдохов подряд (3–6 сек каждый). Пятый выдох — дерево полностью в цвету.
- Инструкция Ляли перед началом: «Вдохни через нос — медленно. Теперь дуй ртом, долго-долго,
  пока листочки не распустятся!»

**Уровни:**
- Easy: выдох ≥3 сек, 5 раундов.
- Medium: выдох ≥4 сек, 6 раундов.
- Hard: выдох ≥5 сек, 7 раундов.

**Метрика успеха:** stableRatio ≥ 0.75 (≥75% тиков выше порога в течение требуемой длительности).

### Упражнение 3. Плавный старт («Начни мягко»)

**Педагогическая цель:** сформировать мягкий голосовой старт — постепенное включение голоса
без резкого напряжения гортани. Прицельно работает с тоническим компонентом заикания,
при котором наибольший риск блока — именно первый звук слова.

**Целевой навык по FS:** soft onset — плавный нарастающий старт амплитуды в начале слова.

**Игровой сценарий:** Ляля зажигает фонарик. Резкий щелчок — лампочка перегорает. Плавный
свет — фонарик светит ярко.

**Методический принцип отбора слов:** слова, начинающиеся с гласных (наименьшее напряжение
при старте) или сонорных Л, М, Н (мягкий старт физиологически проще). Избегаем слов
на взрывных Т, П, К, Б — при тоническом заикании именно эти согласные провоцируют блок,
но для данного упражнения нужен успех, а не трудность. Взрывные вводятся позже, на этапе
автоматизации навыка.

**Контент — слова для Упражнения 3 (начальный список, stage Easy):**

Слова на гласную: арбуз, облако, утка, иголка, эхо, ослик, аист, улитка, осень, утро.
Слова на Л: лампа, луна, лиса, лимон, лодка, лук, лебедь.
Слова на М: мяч, мама, море, молоко, медведь, малина.
Слова на Н: небо, нос, нитка, нога, ножик.

Всего в stub-листе: 32 слова. Полный контент-пак `pack_soft_onset.json` создаётся в Sprint 13.5.

**Механика:**

- Ребёнок слышит аудио-эталон Ляли с плавным мягким стартом (растянутый первый звук, attack
  time 150–250 мс, нарастание RMS от 0 до peak за 200 мс).
- Ребёнок повторяет слово в микрофон.
- Анализ: AVAudioEngine tap записывает первые 500 мс после детекции начала звука.
  `FluencyAnalyzerWorker` вычисляет `attackTime` = время от первого тика выше threshold
  до достижения 80% от peak RMS.

  Классификация:
  - attackTime < 50 мс → hard onset → анимация «лампочка мигнула» → Ляля: «Попробуй ещё раз,
    начни чуть медленнее!»
  - attackTime 50–100 мс → borderline → Ляля: «Почти! Ещё чуть плавнее.»
  - attackTime ≥ 100 мс → soft onset → reward → фонарик загорается ярко.

- 5 попыток на одно слово (после 2 неудачных подряд — Ляля показывает анимацию медленного
  нарастания голоса). 5 слов на сессию.

**Метрика успеха:** ≥3 из 5 слов с soft onset (attackTime ≥ 100 мс) → сессия засчитана.

### Упражнение 4. Дневник речи («Считаем спотыкания»)

**Педагогическая цель:** дать родителю и логопеду объективную метрику динамики заикания
во времени. Ребёнок читает или повторяет текст вслух — система подсчитывает признаки
нарушения плавности. Упражнение не является коррекционным для ребёнка — это инструмент
наблюдения.

**Примечание:** результаты Дневника речи показываются **только в родительском интерфейсе**
(Parent Dashboard). Ребёнку показывается нейтральная анимация «Ляля слушает», без цифр
и оценки.

**Механика:**

- На экране: короткий текст (3–5 предложений, детские темы) или картинка с подписью.
  Ребёнок читает вслух (если умеет) или повторяет вслед за Лялей фразу за фразой.
- AVAudioEngine записывает аудио фразы.
- WhisperKit → транскрипт (offline, русская модель).
- `FluencyAnalyzerWorker` анализирует транскрипт + временну́ю сетку токенов:

  Детектируемые признаки дисфлюэнтности:
  1. **Повтор слога/слова** — в транскрипте подряд идут одинаковые токены («ма-ма-ма», «и и и»).
  2. **Удлинение гласной (prolongation)** — длительность гласной в тайм-сетке WhisperKit
     > 300 мс (норма для детской речи ≤ 200 мс).
  3. **Пауза внутри слова (block)** — разрыв между началом и продолжением слова > 800 мс
     при условии, что соседние слова произнесены без разрыва (эвристика: токены слова
     разделены силенс-фреймом > порога, при этом общий темп речи нормален).

  Метрика: «спотыканий на 100 слогов» (стандартный показатель в русской логопедии).
  Норма: ≤ 5. Лёгкое заикание: 5–15. Среднее: 15–30.

- История по сессиям → Swift Charts, тренд за 4 недели в Parent Dashboard.
- Сессия Дневника: 1 текст, ~2 минуты записи.

**Контент — тексты для stub (5 текстов hardcoded в MVP Sprint 13.5):**

1. Кошка спит на диване. Она мягкая и тёплая. Мама гладит кошку.
2. Машина едет по дороге. Гудит гудок: би-би! Дорога длинная.
3. Котёнок пьёт молоко. Молоко вкусное. Котёнок доволен.
4. Мальчик рисует домик. У домика красная крыша. Рядом растёт дерево.
5. Птичка сидит на ветке. Она поёт песенку. Солнышко светит ярко.

Полный контент-пак `pack_stuttering_texts.json` (20 текстов, теги по сложности и тематике)
создаётся speech-specialist в отдельном sprint. F5 использует 5 hardcoded текстов.

## Структура модуля и VIP-архитектура

Модуль организован как единый раздел `StutteringModule` с 4 sub-mode:

```
Features/StutteringModule/
├── StutteringView.swift              — root view, выбор режима (4 карточки)
├── StutteringInteractor.swift        — координация, маршрутизация по sub-mode
├── StutteringPresenter.swift         — ViewModel для root view
├── StutteringRouter.swift            — навигация
├── StutteringModels.swift            — Request / Response / ViewModel
├── Metronome/
│   ├── MetronomeView.swift
│   ├── MetronomeInteractor.swift
│   └── MetronomeModels.swift
├── BreathingExtended/                — расширение существующего Breathing
│   └── BreathingExtendedInteractor.swift   — subclass/composition BreathingInteractor
├── SoftOnset/
│   ├── SoftOnsetView.swift
│   ├── SoftOnsetInteractor.swift
│   └── SoftOnsetModels.swift
├── FluencyDiary/
│   ├── FluencyDiaryView.swift        — только «Ляля слушает», без цифр
│   ├── FluencyDiaryInteractor.swift
│   ├── FluencyDiaryParentView.swift  — Parent Dashboard chart
│   └── FluencyDiaryModels.swift
└── Workers/
    ├── MetronomeWorker.swift         — AVAudioPlayer tick, Timer-based BPM
    ├── FluencyAnalyzerWorker.swift   — attackTime, prolongation, block detection
    └── SpeechRecordWorker.swift      — запись для FluencyDiary → WhisperKit
```

Ориентировочный LOC по компонентам:
- StutteringView + StutteringInteractor + Presenter + Router + Models: ~200 LOC
- MetronomeView + MetronomeInteractor + MetronomeModels + MetronomeWorker: ~280 LOC
- BreathingExtendedInteractor (delta от BreathingInteractor): ~120 LOC
- SoftOnsetView + SoftOnsetInteractor + SoftOnsetModels: ~250 LOC
- FluencyDiaryView + FluencyDiaryInteractor + FluencyDiaryParentView + FluencyDiaryModels: ~280 LOC
- FluencyAnalyzerWorker + SpeechRecordWorker: ~200 LOC

**Итого: ~1 330 LOC.** Цель — 1 200+ LOC достигается.

## Адаптивная сложность

| Параметр | Easy | Medium | Hard |
|---|---|---|---|
| Метроном BPM | 75 | 90 | 105 |
| Выдох (сек) | ≥3 | ≥4 | ≥5 |
| Raундов за сессию | 5 | 7 | 10 |
| Слоговая сложность (Метроном) | 1–2 | 3 | 4+ |
| AttackTime порог (soft onset) | ≥100 мс | ≥120 мс | ≥150 мс |

`AdaptivePlannerService` повышает уровень, если fluency rate за 3 последних сессии ≥ 80%.
Понижает, если fluency rate < 50% за 2 сессии подряд.

звуков (С → З → Ц → Ш → Ж → Ч → Щ → Л → Р). Доступен через Parent Dashboard или
кнопку «Специальные упражнения» в детском интерфейсе. Разблокируется, если в профиле ребёнка
включён флаг `hasFluencyGoal: Bool` (устанавливается родителем в настройках).

## Этические замечания

1. **Без диагноза.** Приложение не ставит диагноз заикания и не классифицирует его тип или
   степень. В UI используется слово «плавность речи», не «заикание».

2. **Onboarding-предупреждение.** При первом входе в модуль родитель видит экран:
   «Если у ребёнка выраженное заикание — эти упражнения помогут практиковаться дома.
   Но для устойчивого результата важна работа с логопедом. Нужна помощь? → Найти специалиста.»
   Ссылка на специалиста — не внешняя ссылка, а текстовый раздел с рекомендациями.

3. **Метрики — только родителю.** Данные Дневника речи (количество спотыканий, динамика)
   отображаются **исключительно** в Parent Dashboard. В детском UI — только нейтральная
   анимация Ляли без числовой обратной связи.

4. **Позитивный язык в детском UI.** Нет слов «заикание», «неправильно», «снова». Только
   «плавно», «мягко», «ещё разок», «почти получилось!»

5. **Рекомендация по нагрузке.** Модуль рекомендован не чаще 1 раза в день, 10–12 минут.
   После 3 сессий подряд без прогресса — Ляля рекомендует сделать перерыв 1 день и
   обратиться к специалисту.
