## Найденные проблемы

### P1-01 — Онбординг, шаг 2 «Кто пользуется приложением?»: переполнение на iPhone SE
- **Экран:** OnboardingRoleStep (онбординг шаг 2/10).
- **Симптом:** карточка роли «Ребёнок» и кнопка «Далее» отрисованы ниже видимой области экрана и недоступны. Экран НЕ прокручивается. Подтверждено скриншотом — видны только «Родитель» и «Логопед».
- **Причина:** `OnboardingRoleStep` (`OnboardingFlowViewComponents.swift:58`) — `VStack { LyalyaHeroView(size:200); title; subtitle; VStack(3 карточки); Spacer() }` без `ScrollView`. Маскот 200pt + заголовок + подзаголовок + 3 карточки + mascot bubble переполняют экран iPhone SE (667pt). Соседние шаги (childName, goals, sounds) обёрнуты в `ScrollView` — role-шаг пропущен.
- **Влияние:** на самом маленьком поддерживаемом устройстве пользователь не может выбрать роль «Ребёнок» и не видит «Далее».
- **Серьёзность:** P1.
- **Исправление:** обернуть `OnboardingRoleStep` в `ScrollView` (как соседние шаги) либо адаптивно уменьшить `LyalyaHeroView` size на компактных экранах.

### P1-03 — SpecialistHome: таб-бар рендерится во весь экран
- **Экран:** SpecialistHomeView (route `specialistHome`).
- **Симптом:** выбранная вкладка «Дети» в `HSAnimatedTabBar` отрисована как капсула высотой **623pt** (вместо ~44pt). Подтверждено `describe-ui`: кнопка «Дети» frame `x=32 y=28 w=123 h=623`; остальные вкладки — h≈19-21. Гигантская синяя капсула перекрывает половину экрана сверху донизу.
- **Причина (для расследования):** в `SpecialistHomeView` (`SpecialistHomeView.swift:40`) `HSAnimatedTabBar` внутри `ZStack(alignment:.bottom)` — выбранный item / capsule indicator получает неограниченную высоту. ParentHomeView использует тот же компонент и рендерится корректно — нужно сравнить, чем отличается контекст (вероятно `specTabContent` для `.children` = `SpecChildListView()` не ограничивает высоту, либо проблема в самом `HSAnimatedTabBar` selected-item layout).
- **Влияние:** экран специалиста визуально сломан, таб-бар нечитаем.
- **Серьёзность:** P1.
- **Исправление:** ограничить высоту item/indicator в `HSAnimatedTabBar` (фиксированная высота строки таб-бара), либо починить констрейнты в `SpecialistHomeView`.

### P2-02 — childHome: неподставленный спецификатор `%d` в разделе «Задания логопеда»
- **Экран:** ChildHomeView, секция «ЗАДАНИЯ ЛОГОПЕДА».
- **Симптом:** кнопки заданий показывают литеральный текст `«%d задания. Р»` и `«%d задания. Ш»` — спецификатор `%d` не подставлен числом.
- **Причина:** где-то `String(localized: "...%d задания...")` используется без форматирования аргумента (вероятно в accessibility label и/или видимом тексте карточки задания).
- **Влияние:** пользователь видит «%d задания» вместо «2 задания».
- **Серьёзность:** P2 (косметика, не блокирует).
- **Исправление:** найти ключ строки с `%d задания`, применить `String(format:)` / интерполяцию с реальным count.
