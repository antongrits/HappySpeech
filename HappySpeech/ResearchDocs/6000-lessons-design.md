## 1. Общая концепция матрицы

HappySpeech строит 6000+ уроков из конечного набора измерений, перемноженных методологически осмысленными способами. Это не «6000 разных заданий придуманных вручную» — это **контентный движок** (ContentEngine), который собирает урок из:
- целевого звука
- этапа работы
- позиции звука
- шаблона игры
- тематики
- уровня сложности
- набора слов/фраз

**Принцип:** матрица большая, но отфильтрованная. Из теоретических ~20 000 комбинаций методологические фильтры оставляют ~6 200 осмысленных уроков.

## 3. Формула расчёта объёма

### Принцип фильтрации

Теоретический максимум: 20 звуков × 15 этапов × 16 шаблонов × 6 тем × 3 уровня = 86 400 комбинаций.

Реально осмысленные комбинации получаются после применения следующих фильтров:

**Фильтр 1: Совместимость шаблон × этап.**
Не каждый шаблон применим на каждом этапе. Например:
- ART применим только на этапе 0
- RAM применим на этапах 1–12
- MP применим только на этапе 14 (дифференциация)
- NQ применим на этапах 11–13

Матрица совместимости даёт ~40% рабочих комбинаций (вместо 100%).

**Фильтр 2: Минимальный контентный объём.**
Урок требует минимум 6–12 слов/фраз. Для редких звуков в конечной позиции — слов может не хватить. Если контент меньше 6 единиц — урок не создаётся.

**Фильтр 3: Возрастная применимость.**
Некоторые этапы и темы недоступны для возраста 5 лет (звукобуквенный анализ, сложные стечения, длинные предложения).

**Фильтр 4: Методологический смысл.**
Не создаём урок «Р в конечной позиции — шаблон ART»: артикуляционное упражнение не работает со словами.

## 4. Детальная структура каждого урока

Каждый урок (Lesson) — это структурированный объект с фиксированными полями:

```
Lesson {
  lesson_id: String          // уникальный ID: "sh_w_init_L&C_clothes_easy_001"
  target_sound: String       // "Ш"
  sound_group: String        // "sibilant_hissing"
  stage: Int                 // 6 (word_init)
  stage_name: String         // "word_init"
  template: String           // "listen-and-choose"
  theme: String              // "одежда"
  difficulty: Int            // 1 (easy)
  age_min: Int               // 5
  age_max: Int               // 8
  duration_seconds: Int      // 180 (3 мин)
  items: [LessonItem]        // 8–12 элементов
  step_sequence: [Step]      // последовательность шагов
  rewards: RewardConfig      // параметры наград
  audio_model_id: String     // ссылка на аудиоэталон логопеда
  phonetic_notes: String     // методические заметки для персонажа
}
```

### Структура LessonItem

```
LessonItem {
  item_id: String
  text: String               // слово/фраза/предложение
  audio_id: String           // ссылка на аудио
  image_id: String           // ссылка на картинку
  target_sound: String       // "Ш"
  sound_position: String     // "beginning" | "middle" | "end" | "cluster" | "isolated"
  syllable_count: Int        // 1 / 2 / 3 / 4
  difficulty: Int            // 1 / 2 / 3
  categories: [String]       // ["одежда", "2syllables"]
  phonetic_features: {
    stressed_syllable: Int   // номер ударного слога
    target_sound_index: Int  // позиция целевого звука в слове
    nearby_sounds: [String]  // другие «сложные» звуки рядом
  }
  distractors: [String]      // неправильные варианты для L&C, MP
  correct_answer: String     // правильный ответ (для L&C)
}
```

### Последовательность шагов (step_sequence)

Каждый шаблон имеет фиксированную последовательность шагов. Пример для L&C:

```
Step 1: INTRO — персонаж представляет задание (5–8 сек)
Step 2: DEMO — один демонстрационный пример (10–15 сек)
Step 3: PRACTICE × N — основная практика (N = 8–10 карточек)
Step 4: FEEDBACK — итоговая обратная связь (10–15 сек)
Step 5: REWARD — анимация награды (5–10 сек)
```

Для RAM:
```
Step 1: INTRO
Step 2: MODEL — персонаж произносит с акцентом на звук
Step 3: WAIT — VAD ждёт речи ребёнка (3 сек тайм-аут)
Step 4: RECORD — запись ASR
Step 5: SCORE — вывод результата (попал/почти/ещё)
Step 6: REPEAT или NEXT
Step 7: REWARD
```

- Начало: difficulty 1 (успешный разогрев)

- Конец: difficulty 1–2 (приятное завершение)

## 9. Контентные ограничения (методологические фильтры)

### Запрещённые комбинации

1. MP (minimal-pairs) + этапы 0–13 — только на этапе 14 (дифференциация).
2. Скороговорки + этапы 0–12 — только в свободной речи (этап 13).
3. ART + слова/фразы — артикуляционный шаблон не работает с лексикой.
4. Звук Р + изолированный звук (этап 1) + шаблон NQ — нарратив требует слов, этап 1 — только звук.
5. Этапы 12, 13, 14 + возраст 5 лет — слишком рано.

### Правила отбора слов в урок

1. Все слова урока — одна тема (тематическая связность).
2. Все слова урока — одна позиция звука (позиционная чистота).
3. Слова в рамках одного урока — разная слоговая структура (от простой к сложной).
4. Дистракторы для L&C — из той же тематической группы и схожей слоговой структуры (чтобы выбор был осмысленным).
5. Нет слов с другим нарушаемым звуком на ранних этапах (принцип «чистоты стимула»): если работаем над С, избегаем слов с Ш и Р в одном уроке на этапах 2–6.

### Принцип «чистоты стимула»

На этапах 0–6: слова для автоматизации целевого звука не содержат других «трудных» звуков (шипящих, сонор Р, Л).
На этапах 7–10: допустимы 1–2 других согласных умеренной сложности.
На этапах 11–13: без ограничений.

## 10. Алгоритм генерации маршрута AdaptivePlannerService

### Входные данные

```swift
struct UserProfile {
  let userId: String
  let age: Int                          // 5, 6, 7, 8
  let targetSounds: [String]            // ["С", "Ш"]
  let currentStage: [String: Int]       // ["С": 6, "Ш": 2]
  let masteredSounds: [String]          // ["К", "Г"]
  let lastFiveTemplates: [String]       // для антиротации
  let sessionHistory: [SessionResult]   // прогресс по дням
  let fatigueLevel: Int                 // 0 (норма) / 1 (умеренная) / 2 (высокая)
  let preferredThemes: [String]         // любимые тематики
}
```

### Алгоритм построения сессии

**Шаг 1: Определить звук для текущей сессии.**
- Приоритет: звук с наибольшим прогрессом (80% критерий близко к достижению)
- Не более 2 звуков за сессию (если только не дифференциация)

**Шаг 2: Выбрать этапы для двух блоков сессии.**
- Блок 1: текущий этап звука
- Блок 2: предыдущий этап (закрепление) или фонематика (если ФФН/ОНР)

**Шаг 3: Подобрать шаблоны с антиротацией.**
```swift
func pickTemplates(forStage: Int, excluding: [String]) -> [String] {
  let compatible = templateMatrix[forStage]
  let available = compatible.filter { !excluding.contains($0) }
  return Array(available.shuffled().prefix(3))
}
```

**Шаг 4: Выбрать тему.**
- Предпочтительная тема пользователя (если задана)
- Не та же, что использовалась в последних 3 сессиях
- Случайная при отсутствии данных

**Шаг 5: Подобрать слова.**
- ContentEngine выбирает 8–12 слов из контент-пака по критериям:
  - целевой звук
  - этап (позиция)
  - тема
  - сложность (начинаем с 1, заканчиваем 2–3)
  - не повторять слова из последних 2 сессий

**Шаг 6: Сформировать step_sequence.**
- Начало: 1–2 упражнения из предыдущего этапа (warm-up)
- Основная часть: новый материал
- Конец: лёгкие слова/слоги для уверенного завершения

**Шаг 7: Настроить параметры сессии.**
- Длина сессии по возрасту: 5л→7 мин, 6л→10 мин, 7л→12 мин, 8л→14 мин
- Если fatigueLevel=1 → сократить на 20%, убрать RAM
- Если fatigueLevel=2 → только L&C и MEM (пассивные), 5 мин

**Шаг 8: Записать план в DailyPlan.**
```swift
struct DailyPlan {
  let date: Date
  let userId: String
  let sessions: [PlannedSession]        // обычно 1–2 на день
  let spaceRepetitionReview: [String]   // слова из SRS для повторения
  let parentNotification: String        // «Сегодня [имя] учит Р в начале слова»
}
```

### Правила spaced repetition

Слово получает статус «требует повторения» если:
- Было ошибкой в последней сессии → повторить через 1 день
- Было «почти» → повторить через 3 дня
- Было «верно» → повторить через 7 дней
- После 3 подряд «верно» → 14 дней → 30 дней → «выучено»

ContentEngine всегда резервирует 2–3 позиции в уроке для SRS-повторения (из «требует повторения»).
