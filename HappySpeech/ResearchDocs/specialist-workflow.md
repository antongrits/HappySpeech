## 1. Роль специалиста в HappySpeech

Специалистский режим открывается для пользователей с ролью «логопед» или «специалист». Он предоставляет расширенный доступ к:
- Подробным данным о каждой сессии ребёнка
- Инструментам диагностики и планирования
- Гибкой настройке маршрута коррекции
- Экспорту данных для отчётов и ведения документации

**Позиция HappySpeech в специалистском контексте:**
Приложение — инструмент для домашней практики между сессиями с живым логопедом. Специалист ставит звук, определяет маршрут, контролирует прогресс. HappySpeech обеспечивает регулярность: ребёнок занимается дома каждый день, и специалист видит все результаты онлайн.

Это не замена специалиста, а мощное дополнение: вместо 2–3 занятий в неделю на практике (у логопеда) + 0 дома → специалист + 5 занятий в неделю дома через HappySpeech = значительно более быстрый прогресс.

## 2. Первичная диагностика

### 2.1 Диагностический профиль в приложении

Специалист заполняет диагностическую карту:

**Блок 1: Общая информация**
- Имя, возраст, дата начала работы
- Тип учреждения (ДОУ, школа, частная практика, домашняя)
- Характер обращения (по инициативе родителя / по направлению врача / плановая логопедическая работа в ДОУ)

**Блок 2: Логопедическое заключение**
Специалист выбирает из стандартизованного списка:
- Дислалия (функциональная / механическая)
- ФФН
- ОНР (I / II / III / IV уровень)
- ЗРР
- Логоневроз / заикание
- Лёгкая дизартрия (псевдобульбарная / другое)
- Другое (свободное поле)

**Блок 3: Звуковой статус**
Для каждого звука: норма / нарушение типа / пропуск / замена на [что].

Пример заполнения:
```
С — искажение (межзубный)
З — замена на С
Ш — норма
Р — пропуск
Л — замена на В (двугубный)
```

**Блок 4: Фонематический слух**
Уровень по шкале ФС 1–5 (описана в speech-methodology-full.md).

**Блок 5: Грамматика и связная речь**
Краткая оценка по шкале:
- Словарь: возраст-нормативный / ниже нормы / значительно ниже
- Грамматика: норма / единичные ошибки / стойкий аграмматизм
- Связная речь: норма / затруднена / недоступна

**Блок 6: Результаты аудиометрии**
Ссылка на результаты или поле «норма» / «снижение» / «не обследован».

### 2.2 Приоритизация звуков

На основании диагностического профиля специалист расставляет приоритеты:

| Уровень | Звуки |
|---------|-------|
| P1 — работаем сейчас | 1–2 звука |
| P2 — следующие | 2–3 звука |
| P3 — после P2 | остальные |

Принцип приоритизации:
1. Более ранние по онтогенезу — приоритетнее
2. Звуки, влияющие на школьную готовность — приоритетнее
3. Не более 2 звуков одновременно (из разных артикуляционных групп)

## 3. Составление индивидуального плана

### 3.1 Краткосрочный план (на 4 недели)

Специалист задаёт:
- Целевые звуки на период
- Этапы (начальный и финальный для каждого звука)
- Частота занятий (приложение адаптирует напоминания)
- Акценты (например, «больше фонематических игр», «меньше RAM — ребёнок устаёт»)

### 3.2 Долгосрочный план (3, 6, 12 месяцев)

Ориентировочный маршрут по всем нарушениям:

**Пример: Дислалия (С межзубный, Ш норма, Р пропуск). Возраст 6 лет.**

| Период | Звук | Этап | Ориентировочный результат |
|--------|------|------|--------------------------|
| Месяц 1 | С | 0–3 (арт.подготовка → обратные слоги) | Правильный изолированный С, слоги |
| Месяц 2 | С | 4–7 (слова начало/середина) | С в словах (начало, середина) |
| Месяц 3 | С | 8–11 (слова конец → предложения) | С автоматизирован в словах |
| Месяц 4 | С + Р | 12–14 (С: дифф/текст) + 0–1 (Р: арт.подготовка) | С в свободной речи + начало работы над Р |
| Месяц 5–6 | Р | 1–7 (изол. → слова начало) | Изолированный Р, слоги, начало слова |
| Месяц 7–9 | Р | 8–13 (позиции → свободная речь) | Р автоматизирован |
| Месяц 10–12 | Р/Л + дифф. | 14 | Дифференциация Р/Л, если актуально |

**Важно:** Это ориентир, не жёсткий план. Прогресс зависит от регулярности занятий, индивидуальных особенностей, участия родителей. Специалист пересматривает план ежемесячно.

### 3.3 Редактирование маршрута

Специалист может:
- Перевести ребёнка вперёд/назад по этапам вручную
- Добавить/убрать шаблоны из ротации
- Заблокировать определённые типы заданий (например, убрать все задания с таймером)
- Добавить специфические слова в «приоритет» (слова, которые ребёнок использует в жизни, но произносит неправильно)

## 4. Ежедневная работа специалиста

### 4.1 Инициация занятия в кабинете логопеда

Специалист может использовать HappySpeech прямо на сессии:
- Выбрать конкретный урок или шаблон (не через автоматический план)
- Задать демонстрацию для ребёнка на экране
- Использовать AR-режим как «логопедическое зеркало»
- Сравнить запись ребёнка с аудиоэталоном в реальном времени

### 4.2 Фиксация ручных наблюдений

После очного занятия специалист добавляет заметку:
- Что отработано
- Наблюдения («при Р появляется носовой оттенок»)
- Рекомендации для домашней практики
- Корректировка плана

Заметки видны в родительском разделе (если специалист выбрал «показать родителям»).

### 4.3 Назначение конкретного домашнего задания

Специалист может сформировать «задание на неделю»:
- Конкретные уроки (из матрицы)
- Конкретные слова для домашней практики
- Игровые задания без приложения (из списка parent-guidance)

Родитель видит задание в разделе «Задание логопеда».

## 5. Мониторинг прогресса

### 5.1 Дашборд специалиста

Экран специалиста показывает:

**Общий обзор:**
- Количество сессий за неделю / месяц
- Средний процент верных ответов (по дням)
- Текущий этап для каждого целевого звука

**Детальная аналитика:**
- Тепловая карта ошибок: слово × сессия × процент ошибок
- Наиболее «трудные» слова (топ-10 по ошибкам)
- Наиболее лёгкие слова (для оценки, что уже освоено)
- Сравнение этой недели с предыдущей

**Аудиозаписи:**
- Лучшая запись произношения за неделю для каждого целевого звука
- Сравнение «сессия 1» и «последняя сессия» (наглядный прогресс)
- Записи, помеченные системой как «проблемные»

### 5.2 Алерты и уведомления

Специалист получает push или email уведомление:
- Ребёнок не занимался 5+ дней → «требуется напоминание родителям»
- Регресс > 20% за неделю → «рекомендуется пересмотреть план»
- Достижение критерия перехода на следующий этап → «готов к переходу, подтвердите»
- Нетипичный паттерн ошибок → «обратите внимание, возможно нужна коррекция плана»

Переход на следующий этап по умолчанию требует подтверждения специалиста (если специалистский профиль активен). Это предотвращает преждевременный переход.

## 6. Срезы и контрольные точки

### 6.1 Ежемесячный срез

Через 4 недели приложение генерирует «Отчёт за месяц»:

**Раздел 1: Посещаемость**
- % выполнения плана занятий (n из N рекомендованных)
- Среднее число занятий в неделю

**Раздел 2: Прогресс по звукам**
Для каждого целевого звука:
- Этап в начале периода
- Этап в конце периода
- Количество сессий на каждом этапе
- Средний процент верных ответов

**Раздел 3: Аудиодинамика**
- 2 аудиофрагмента: первая и последняя сессия месяца
- Визуализация спектра (если VA включена)

**Раздел 4: Рекомендации**
Автоматически генерируемые рекомендации на следующий месяц.

### 6.2 Финальный срез (завершение работы)

При завершении работы над звуком (этап 13 пройден):

**Сертификат** (игровой, для ребёнка): «[Имя] научился говорить звук [Ш]!»

**Отчёт для специалиста:**
- Общая длительность работы (дни/недели/месяцы)
- Число сессий
- Динамика по этапам (графическая)
- Аудиофрагменты «до» и «после»

**Отчёт для родителей:**
- Простое, без жаргона описание достижения
- Следующие шаги (если есть другие звуки)

## 7. Работа с несколькими профилями

### Для логопеда ДОУ

Логопед детского сада ведёт несколько детей одновременно (типично: 10–25 детей в ставке). Специалистский раздел поддерживает:
- Просмотр всех профилей в списке с фильтрами (по группе, по звуку, по статусу)
- Массовые уведомления родителям одной группы
- Сравнительная аналитика (анонимная) по группе — для отчёта
- Быстрое переключение между профилями

### Для частного логопеда

- Календарь занятий с привязкой к профилям детей
- Автоматическое создание «домашнего задания» после очного занятия
- Отслеживание оплаты занятий (базовый учёт)

### Конфиденциальность

Данные каждого ребёнка закрыты. Специалист видит только профили детей, к которым у него есть доступ (выдаётся родителем через код приглашения). Родитель в любой момент может отозвать доступ специалиста.

## 8. Коммуникация с родителями

### 8.1 Рекомендации в приложении

Специалист создаёт рекомендацию — она появляется в родительском разделе:
- «На этой неделе сосредоточьтесь на словах с Р в начале»
- «Попробуйте игру "найди в комнате" (см. руководство для родителей)»
- «[Имя] устаёт во второй половине сессии — попробуйте заниматься утром»

### 8.2 Сообщения в приложении

Специалист может отправить текстовое сообщение родителю через безопасный канал внутри приложения. История сообщений сохраняется.

### 8.3 Готовые шаблоны

Специалист выбирает из готовых шаблонов сообщений (рекомендации, коррекции плана, напоминания) и адаптирует под конкретного ребёнка.

## 9. Экспорт и отчёты

### Форматы экспорта

- **PDF** — для распечатки, ведения карточки в бумажном виде, передачи в школу
- **CSV** — для обработки данных в Excel (статистика, грантовые отчёты)

### Экспортируемые данные

**Отчёт для ПМПК (психолого-медико-педагогическая комиссия):**
- Диагностический профиль (дата начала, заключение)
- Динамика по звукам (таблица)
- Количество часов занятий
- Итоговая оценка

**Отчёт для родителей (за период):**
- Простым языком, без жаргона
- Что выучили, что осталось
- График посещаемости

**Технический отчёт (для специалиста):**
- Все данные сессий в CSV
- Аудиозаписи (ZIP-архив по датам)
- Спектральные данные (если VA использовалась)

## 10. Специализированные настройки

### Настройка шаблонов

Специалист может:
- Включить/выключить конкретные шаблоны для конкретного ребёнка
- Задать обязательные шаблоны (например, «каждая сессия должна начинаться с ART»)
- Задать темп эталона в RAM (0.75×, 1×, 1.25×)
- Настроить пороги перехода (по умолчанию 80%; для данного ребёнка — 70% или 90%)

### Настройка интерфейса

- Включить/выключить таймер (при заикании — выключить)
- Режим «без соревнования» (убрать рейтинг, очки, сравнение)
- Увеличить шрифт и элементы (для детей с нарушениями зрения)
- Выбрать «голос персонажа» (мужской/женский; темп)

### Настройка ASR

Специалист может скорректировать порог скоринга:
- «Строгий» (90% совпадение с эталоном = «верно»)
- «Стандартный» (75%)
- «Мягкий» (60%) — для детей с дизартрией в начале работы

## 11. Случаи для перенаправления

Специалист HappySpeech должен рекомендовать очную консультацию к другим специалистам:

**К неврологу:**
- Подозрение на дизартрию (мышечный тонус, слюнотечение, нарушение глотания)
- Заикание впервые появившееся или ухудшающееся
- Регресс речи после нормального развития

**К отоларингологу:**
- Подозрение на нарушение слуха
- Хроническое нарушение носового дыхания
- Укороченная уздечка языка (не позволяет выполнять упражнения для Р/Л)

**К дефектологу/психологу:**
- ОНР I–II уровня — требуется комплексная коррекция
- ЗРР в сочетании с поведенческими трудностями
- Подозрение на РАС (расстройство аутистического спектра)

**К ортодонту:**
- Выраженный неправильный прикус (препятствует постановке свистящих/шипящих)
- Короткая уздечка языка (дооперационная консультация)

**Важно:** специалист HappySpeech — это педагогический, а не медицинский специалист. Медицинские решения (операция уздечки, медикаменты, диагностика ОНР) — в компетенции врачей-специалистов.

---

*Специалистский раздел HappySpeech проектируется совместно с практикующими логопедами ДОУ и частной практики. Все функции проходят пилотное тестирование с 5+ специалистами до выхода в релиз.*
